# 003. Левое меню и навигация — бизнес-требования

## 1. Цель

Привести левое меню приложения `TK.Tasks` к виду референса и сделать его более полезным для пользователя: улучшить визуальное состояние активного пункта, добавить быстрый доступ к категориям задач и показывать счётчики задач рядом с релевантными пунктами меню.

Дополнительно логотип и текст бренда должны работать как переход на главную страницу приложения.

## 2. Область изменений

Фича затрагивает левое меню приложения и поведение навигации внутри продукта.

Пользователь должен быстрее понимать своё текущее местоположение в приложении, видеть количество актуальных задач и быстрее переходить к нужной группе задач.

## 3. Пользовательские сценарии

### 3.1. Переход на главную через бренд

Пользователь может кликнуть на область бренда в левом меню:

- иконку логотипа;
- текст `TK.Tasks`;
- подпись `Личный планировщик`.

После клика пользователь должен перейти на главную страницу приложения — экран `Обзор`.

### 3.2. Навигация по основным разделам

Пользователь видит в левом меню основные пункты навигации:

- `Обзор`;
- `Мои задачи`;
- `Календарь`;
- `Важное`, если этот пункт требуется для соответствия референсу.

При выборе пункта пользователь переходит в соответствующий раздел.

Активный пункт должен быть визуально выделен в стиле референса, чтобы пользователь сразу понимал, где он находится.

### 3.3. Просмотр счётчиков в меню

Для пунктов меню, где счётчик помогает оценить объём работы, рядом с названием должен отображаться компактный бейдж.

Счётчики должны быть динамическими и обновляться при изменении задач.

Минимально ожидаемые счётчики:

| Пункт | Что показывает |
| --- | --- |
| `Мои задачи` | количество невыполненных задач на сегодня |
| `Важное` | количество важных задач |

Если счётчик равен `0`, он может быть скрыт для основных пунктов меню, как в референсе.

### 3.4. Просмотр категорий

Под основным меню пользователь видит отдельный блок `Категории`.

В блоке отображаются все поддерживаемые категории задач:

| Категория | Отображение |
| --- | --- |
| `Работа` | цветной маркер категории и счётчик |
| `Личное` | цветной маркер категории и счётчик |
| `Здоровье` | цветной маркер категории и счётчик |
| `Обучение` | цветной маркер категории и счётчик |
| `Дом` | цветной маркер категории и счётчик |

Счётчик категории должен показывать количество невыполненных задач в этой категории.

### 3.5. Переход по категории

Пользователь может кликнуть по категории в левом меню.

После клика пользователь должен попасть на страницу задач с применённым фильтром выбранной категории.

Ожидаемое поведение:

- список задач показывает задачи выбранной категории;
- пользователь может визуально понять, что категория выбрана;
- поиск и сортировка на странице задач продолжают работать.

## 4. Визуальные требования

### 4.1. Активный пункт меню

Активный пункт меню должен соответствовать референсу:

- фон активного пункта темнее/контрастнее обычного;
- слева отображается вертикальный золотой индикатор;
- иконка активного пункта подсвечивается золотым цветом;
- текст активного пункта становится более контрастным;
- счётчик активного пункта получает золотой акцент.

### 4.2. Счётчики

Счётчики должны выглядеть как компактные бейджи:

- расположены справа от названия пункта;
- имеют округлую форму;
- не ломают выравнивание меню;
- не занимают лишнее место при коротких и длинных названиях.

### 4.3. Категории

Категории должны быть визуально отделены от основного меню заголовком `Категории`.

Каждая категория должна иметь:

- цветной круглый маркер;
- название;
- счётчик невыполненных задач.

## 5. Бизнес-правила

1. Счётчики должны рассчитываться по актуальному списку задач.
2. Выполненные задачи не должны попадать в счётчики `Мои задачи` и категорий.
3. Счётчик `Важное` должен учитывать задачи с отметкой важности.
4. Если задача меняет статус выполнения, счётчики должны обновляться.
5. Если задача создаётся или удаляется, счётчики должны обновляться.
6. Если задача меняет категорию, счётчики категорий должны обновляться.
7. Клик по бренду всегда ведёт на главную страницу.

## 6. Нефункциональные требования

### 6.1. Понятность

Меню должно оставаться простым и быстро считываемым. Добавление категорий и счётчиков не должно визуально перегружать левую панель.

### 6.2. Консистентность

Новые элементы должны соответствовать текущей дизайн-системе приложения и референсу:

- тёмная тема;
- золотые акценты;
- округлые элементы;
- существующие цвета категорий.

### 6.3. Адаптивность

Изменения левого меню не должны ухудшать работу приложения на мобильных устройствах.

Если на мобильной версии левое меню не отображается, новые категории и счётчики могут быть доступны только в расширенной версии меню на больших экранах.

## 7. Acceptance criteria

Фича считается готовой, если:

1. Активный пункт меню визуально соответствует референсу.
2. В левом меню есть блок `Категории`.
3. Все категории отображаются с цветным маркером и счётчиком.
4. Пункты меню с релевантными задачами отображают счётчики.
5. Счётчики обновляются после выполнения, создания и удаления задач.
6. Клик по категории открывает список задач с фильтром этой категории.
7. Клик по логотипу или тексту бренда ведёт на главную страницу.
8. Мобильная навигация не сломана.
