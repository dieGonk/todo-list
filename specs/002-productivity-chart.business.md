# 002. График продуктивности — бизнес-требования

## 1. Цель

Реализовать на главном экране график продуктивности, чтобы пользователь мог быстро оценить свою динамику выполнения задач за последнюю неделю.

График должен помочь пользователю понять:

- насколько продуктивной была текущая неделя;
- в какие дни было выполнено больше всего задач;
- сколько задач сейчас находится в работе;
- сколько задач уже завершено;
- сколько задач ещё предстоит выполнить.

## 2. Пользовательская ценность

Пользователь получает наглядный обзор своей активности без необходимости вручную анализировать список задач.

График должен мотивировать пользователя поддерживать регулярность выполнения задач и помогать замечать дни с высокой или низкой продуктивностью.

## 3. Место в продукте

График продуктивности отображается на экране `Обзор` / dashboard.

Он является частью блока аналитики рядом с другими виджетами прогресса и задач на сегодня.

## 4. Основной сценарий

1. Пользователь открывает приложение.
2. Пользователь попадает на экран обзора.
3. На экране отображается карточка `Продуктивность`.
4. Пользователь видит динамику выполненных задач за последние 7 дней.
5. Пользователь видит день с максимальным количеством выполненных задач.
6. Пользователь видит краткие показатели по задачам:
   - в работе;
   - завершено;
   - предстоит.

## 5. Период отображения

На первом этапе график показывает данные за последние 7 календарных дней.

Пользователь не выбирает период вручную. Переключение периодов можно добавить позже.

## 6. Что считается продуктивностью

Продуктивность за день — это количество задач, которые пользователь завершил в этот день.

Если за день не было завершённых задач, этот день должен отображаться с нулевым значением.

## 7. Метрики карточки

Карточка должна содержать три кратких показателя.

### 7.1. В работе

Количество задач, которые ещё не завершены.

### 7.2. Завершено

Количество задач, которые уже завершены.

### 7.3. Предстоит

Количество незавершённых задач, запланированных на будущие даты.

## 8. Визуальные требования

График должен соответствовать визуальному стилю дизайн-референса.

Карточка должна содержать:

- заголовок `Продуктивность`;
- пометку периода `Последние 7 дней`;
- три краткие метрики;
- плавный график с заливкой области;
- подписи дней недели;
- выделение дня с максимальной продуктивностью;
- подпись над максимальной точкой, например `5 задач`.

## 9. Поведение при отсутствии данных

Если за последние 7 дней пользователь не завершал задачи:

- график всё равно должен отображаться;
- все значения должны быть нулевыми;
- интерфейс не должен выглядеть сломанным или пустым.

## 10. Поведение при ошибке загрузки данных

Если данные графика временно недоступны, экран обзора должен оставаться доступным.

Внутри карточки можно показать короткое сообщение:

```txt
Не удалось загрузить статистику
```

## 11. Адаптивность

График должен корректно отображаться:

- на desktop;
- на планшетах;
- на мобильных устройствах.

На маленьких экранах график может занимать всю ширину карточки и становиться компактнее, но должен оставаться читаемым.

## 12. Ограничения первого этапа

На первом этапе не требуется:

- выбор произвольного периода;
- сравнение недель;
- экспорт статистики;
- детальная аналитика по категориям;
- персональные рекомендации.

Эти возможности могут быть добавлены позже.

## 13. Acceptance criteria

Фича считается реализованной, если:

1. На экране обзора есть карточка `Продуктивность`.
2. Карточка показывает данные за последние 7 дней.
3. Карточка показывает метрики `В работе`, `Завершено`, `Предстоит`.
4. График показывает количество завершённых задач по дням.
5. День с максимальной продуктивностью визуально выделен.
6. Под графиком отображаются подписи дней недели.
7. Если данных нет, график отображается с нулевыми значениями.
8. Если данные недоступны, пользователь видит понятное сообщение об ошибке.
9. График визуально соответствует дизайн-референсу.
10. График корректно отображается на мобильных устройствах.
