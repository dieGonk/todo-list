# 004. Форма создания задачи — бизнес-требования

## 1. Цель

Привести форму создания задачи и связанные элементы управления списком задач к единому визуальному стилю референса.

Пользователь должен быстро понимать, где вводить новую задачу, как выбрать категорию и приоритет, а также как управлять сортировкой списка задач.

## 2. Область изменений

Фича затрагивает экран списка задач:

- форму добавления новой задачи;
- кнопку `Добавить`;
- выбор категории;
- выбор приоритета;
- выбор сортировки списка задач.

## 3. Пользовательские сценарии

### 3.1. Создание задачи

Пользователь открывает список задач и видит форму добавления задачи в верхней части экрана.

Форма должна быть визуально отделена от списка задач и выглядеть как единый аккуратный блок.

Пользователь может:

- ввести название задачи;
- выбрать категорию;
- выбрать приоритет;
- нажать кнопку `Добавить`;
- добавить задачу клавишей Enter, если название заполнено.

### 3.2. Кнопка добавления

Кнопка `Добавить` должна выглядеть как основной призыв к действию.

Ожидаемое поведение:

- кнопка заметна визуально;
- кнопка использует золотой акцент;
- кнопка неактивна, если название задачи пустое;
- неактивное состояние должно быть визуально понятно.

### 3.3. Выбор категории

Пользователь может выбрать категорию задачи из выпадающего списка.

Элемент выбора категории должен визуально соответствовать остальным контролам формы и не выбиваться из общего стиля.

### 3.4. Выбор приоритета

Пользователь может выбрать приоритет задачи из выпадающего списка.

Элемент выбора приоритета должен выглядеть так же аккуратно и консистентно, как выбор категории.

### 3.5. Сортировка списка

Пользователь может изменить сортировку списка задач.

Выпадающий список сортировки должен соответствовать стилю фильтров и формы, чтобы экран выглядел цельно.

## 4. Визуальные требования

### 4.1. Форма создания

Форма должна соответствовать референсу:

- единый контейнер с тёмным фоном;
- тонкая граница;
- скруглённые углы;
- комфортные внутренние отступы;
- иконка плюса слева в отдельном пунктирном квадрате;
- визуальный акцент при фокусе внутри формы.

### 4.2. Кнопка `Добавить`

Кнопка должна:

- быть золотой;
- иметь мягкий градиент;
- быть скруглённой;
- содержать иконку плюса и текст;
- иметь одинаковую высоту с полем ввода.

### 4.3. Выпадающие списки

Выпадающие списки должны:

- быть одного стиля между собой;
- соответствовать тёмной теме;
- иметь границу и скругление;
- быть сопоставимыми по высоте с другими контролами;
- иметь понятное состояние фокуса;
- не выглядеть как системные элементы, выбивающиеся из дизайна.

### 4.4. Сортировка

Выбор сортировки должен выглядеть как компактный управляющий элемент в строке фильтров.

Он не должен визуально конфликтовать с кнопками фильтров.

## 5. Бизнес-правила

1. Новую задачу нельзя добавить без названия.
2. Пустое или состоящее только из пробелов название считается пустым.
3. После успешного добавления поле названия очищается.
4. Выбранные категория и приоритет используются при создании задачи.
5. Изменение сортировки меняет порядок отображения задач.
6. Визуальные изменения не должны ухудшать доступность формы.

## 6. Нефункциональные требования

### 6.1. Консистентность

Форма, кнопки и выпадающие списки должны выглядеть как часть одной дизайн-системы.

### 6.2. Понятность

Пользователь должен без подсказок понимать:

- куда вводить название задачи;
- где выбрать категорию;
- где выбрать приоритет;
- какая кнопка создаёт задачу.

### 6.3. Адаптивность

Форма должна оставаться удобной на узких экранах.

Если элементы не помещаются в одну строку, они могут переноситься, но должны сохранять читаемость и удобство нажатия.

## 7. Acceptance criteria

Фича считается готовой, если:

1. Форма создания задачи визуально соответствует референсу.
2. Кнопка `Добавить` соответствует референсу и выглядит как основной action.
3. Выпадающие списки категории и приоритета соответствуют стилю формы.
4. Выпадающий список сортировки соответствует стилю фильтров.
5. Неактивное состояние кнопки `Добавить` понятно пользователю.
6. Создание задачи через кнопку и Enter продолжает работать.
7. Сортировка списка задач продолжает работать.
8. Форма остаётся удобной на мобильных экранах.
